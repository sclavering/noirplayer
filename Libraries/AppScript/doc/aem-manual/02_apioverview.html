Classes

The main aem classes are as follows:

AEMApplication
Represents a scriptable application, and provides methods for constructing AEMEvent instances.
AEMEvent
Represents an Apple event, and provides methods for adding parameters and attributes, and for sending it.
AEMCodecs
Provides -pack: and -unpack: methods for converting Foundation values to NSAppleEventDescriptors, and vice-versa. Clients usually don't need to access this class directly.
AEMType, AEMEnum
Represent Apple event type and enumerator values.

In addition, there are a number of classes used to represent application references, although the user does not instantiate these directly.

Macros

The AEM module exports three macros for use in constructing application references:

AEMApp
Returns the base object used to construct absolute references.
AEMCon
Returns the base object used to construct relative reference to container object (used in by-range specifiers).
AEMIts
Returns the base object used to construct relative reference to object being tested (used in by-filter specifiers).

References are constructed from these base objects using chained property/method calls.

Functions

The aem package also exports several functions:

AEMFindApplication
Locates applications by name, bundle id and/or creator code. (See also the AEMFindAppWithCreator, AEMFindAppWithBundleID and AEMFindApplication macros.)
AEMIsApplicationRunning
Checks if a given application is running.
AEMLaunchApplication
Launches an application (basically a convenience wrapper around the Process Manager's LaunchApplication function).
